Wearable Swimming Motion Data Acquisition System Based on XIAO and JY901

After preliminary discussions with the instructor and defining the research focus, we started the system design.
To collect IMU data during swimmers’ training, the device needs to be compact and low-power. Therefore, we abandoned
complex Wi-Fi real-time data transmission and instead adopted a method where IMU data are written to an SD card. After
the athlete finishes swimming, the SD card can be removed from the device and the data stored on a PC for further analysis.

1. Project Overview

1.1 Project Name

Wearable Swimming Motion Data Acquisition System Based on XIAO and JY901.

1.2 Project Objective

To design and implement a low-cost, portable, and high-precision wearable data acquisition system for real-time capture
of swimmers’ full-body motion data during training. The system uses an inertial measurement unit to collect raw data,
stores it locally on an SD card, and provides data support for subsequent swimming technique analysis, rhythm assessment,
and posture optimization.

1.3 Core Features

  • 9-axis motion data recording: Records 3-axis acceleration, 3-axis angular velocity, and 3-axis magnetic field data.
  • Attitude calculation: JY901’s built-in algorithm outputs stable roll, pitch, and yaw angles.
  • Local data storage: Saves data files to an SD card to ensure completeness in underwater environments where wireless
    transmission is impractical.
  • Simple and reliable control: A physical button allows easy start/stop control, convenient for swimmers to operate in water.

2. System Architecture

This system adopts a distributed acquisition and centralized storage architecture. Each independent node is a complete data
acquisition unit.

2.1 System Block Diagram

2.2 Data Flow

  1. Data sensing: The JY901 IMU collects 9-axis data and performs attitude calculation at a preset frequency (e.g., 100Hz).
  2. Data transmission: The XIAO board reads data from the JY901 via UART serial interface.
  3. Data packaging: XIAO adds timestamps (e.g., via RTC).
  4. Data storage: The packaged data are written in binary format to a file on the SD card.

3.1 Core Controller: Seeed Studio XIAO

  • Reason for Selection: Ultra-small, low-power, sufficient performance, supports UART/I2C, compatible with SD card module.
  • Main Functions:
    • Communicate with JY901.
    • Manage SD card for data writing and file creation.
    • Handle button input to control recording.

3.2 Inertial Measurement Unit: JY901

  • Reason for Selection: Integrated attitude algorithm, plug-and-play, low cost, adequate precision.
  • Interface: UART serial, connected to XIAO via RX/TX pins.
  • Output Data: Provides acceleration, gyroscope, magnetic field data, and calculated Euler angles.

3.3 Storage Module: XIAO SD Card Adapter

  • Reason for Selection: Plug-and-play design, compatible with XIAO pinout.
  • File Format: Data stored in .csv, easy to process using Excel, Python, or MATLAB.
  • Power: Shares the same lithium battery with XIAO, supporting portable use.

3.4 Auxiliary Components

  • Waterproof Enclosure: 3D-printed case for underwater protection.
  • Mounting Strap: Attaches device securely to body.
  • Button: Start/stop recording control.

4.3 Key Implementation Details

  • Data Parsing: Extract and decode IMU data frames according to JY901 protocol.
  • File Management: Create new file each power-up to avoid overwriting old data.
  • Write Optimization: Use dataFile.flush() periodically to prevent data loss.

5. Data Acquisition Scheme

5.1 Sensor Placement

  • Node 1 (Torso): On the waist/back, for body rotation stability.
  • Node 2 (Arm): On the forearm/wrist, for stroke frequency and path.
  • Node 3 (Leg): On calf/ankle, for kick amplitude and rhythm.

5.2 Data Recording Process

  1. Charge all nodes, clear SD cards, attach devices.
  2. Start recording simultaneously via button.
  3. Swim training session, data continuously recorded.
  4. Stop recording after session.
  5. Retrieve SD cards and analyze data on PC.

6. Expected Results and Outputs

  • Hardware Outcome: Fully functional waterproof sensor nodes.
  • Data Outcome: Timestamped multi-dimensional motion datasets (.csv files).
  • Analytical Value:
    • Stroke Rhythm: Calculate stroke frequency via wrist acceleration periodicity.
    • Body Posture: Assess body roll and stability via torso attitude angles.
    • Symmetry: Compare left/right limbs for motion balance.

7. Future Work and Outlook

  • Algorithm Development: Python scripts for stroke rate, distance, and joint analysis.
  • Real-time Feedback: Add Bluetooth for quick data transfer to mobile app.
  • Power Optimization: Add sleep mode to extend battery life.
